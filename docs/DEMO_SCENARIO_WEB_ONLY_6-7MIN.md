# SIT Copilot Web限定デモ台本（6-7分）

## 0. 目的

この台本は、ローカルWeb環境で以下を満たすための収録用ドキュメントです。

- 初期状態（セッション0件）から開始する
- Session A のタイトル自動生成を見せ場として実演する
- Session B で主要機能を紹介する
- Webで公開されている画面操作のみで完結する

## 1. 制約と公開範囲

- 使用ルートは次に限定する
  - `/`
  - `/lectures`
  - `/lectures/:id/live`
  - `/lectures/:id/sources`
  - `/settings`
- 使用しないルート
  - `/procedure`
  - `/readiness-check`
  - `/lectures/:id/qa`
  - `/lecture/:session_id/qa`
- API直叩き、DB確認、CLI出力の画面提示はしない

## 2. 収録前チェック（必須）

1. ローカルWeb環境を起動する。
2. 新規ブラウザプロファイルまたはシークレットウィンドウで開く。
3. `/lectures` でセッションカードが0件であることを確認する。
4. マイク利用を許可する。
5. 録画は1タブ運用で行う。

## 3. 重要な実装依存ルール

- タイトル自動生成トリガーは、`/lectures/:id/live` から画面内の「講義一覧」ボタンで戻ったときのみ有効。
- ブラウザの戻るボタンで戻ると、意図したトリガーにならない可能性がある。
- Aのタイトル候補が出る前にAの手動タイトル編集はしない（自動生成の見せ場を壊すため）。

## 5. A/B実演で使う発話サンプル

### Session A（タイトル生成用）

1. 今日は外れ値と残差プロットの確認方法を扱います。
2. 回帰分析では外れ値が直線を大きく動かすことがあります。
3. 次回課題は scikit-learn で分類モデルを作ることです。

### Session B（Live実演用）

1. ここから機能デモ用の発話を行います。
2. 30秒要約と用語抽出の更新を確認します。
3. 引用付きの質問応答をこのあと実行します。

## 6. タイトル生成見せ場の運用ルール

1. Aから戻るときは必ず画面内「講義一覧」ボタンを使う。
2. Aカードが「タイトル作成中...」になったことを口頭で確認する。
3. 生成待機中はBカード操作を進め、無操作時間を作らない。
4. A候補が出たら1つ選択し、Aタイトルの更新を明示する。

## 7. 失敗時リカバリ（Web操作のみ）

### Aが「タイトル作成中...」にならない

1. Aを開き直す。
2. 発話を1-2行追加する。
3. 画面内「講義一覧」ボタンで戻る。

### A候補が出るのが遅い

1. Bカード操作（フィルタ、コピー、タイトル編集）を先に進める。
2. `/settings` の操作を先に実演しない（本編順序が崩れるため）。
3. A候補表示を確認後に候補適用へ進む。

### Liveで字幕が増えにくい

1. 録音ONを再確認する。
2. マイク権限を再許可する。
3. 1文ずつ区切って発話する。

## 8. 受け入れ条件（チェックリスト）

- [ ] 収録開始時に `/lectures` のセッションが0件である
- [ ] Aで「タイトル作成中...」→候補表示→候補適用を動画内で確認できる
- [ ] A生成待機中にBカードで2種類以上の操作を実演している
- [ ] 非公開ルート（`/procedure`, `/readiness-check` など）を使っていない
- [ ] 無操作説明が連続15秒を超えていない

## 9. 収録後に残すメモ

- Aタイトル候補の表示までの体感時間（秒）
- Bで使用した根拠あり質問 / 根拠不足質問
- つまずいた操作（次回短縮対象）
