# SIT Copilot ポスター発表 3分台本

## 0:00-0:20 タイトルと結論
- 「SIT Copilotは、講義中の聞き漏らしと講義後の復習を支援する学習支援システムです。」
- 「強みは字幕変換の低遅延で、`0.76s`（ja→en）と`0.99s`（ja→easy-ja）を達成しました。」
- 「一方で、QA E2E遅延は`9.18s`で、目標`<5s`は未達です。」

## 0:20-0:50 背景と目的
- 「講義中は聞き漏らし、講義後は根拠追跡が難しいという課題があります。」
- 「対象は留学生、聞き漏らしが多い学生、アクセシビリティ配慮が必要な受講者です。」
- 「目的は、聞き漏らしと非母語話者の理解負荷を下げることです。」

## 0:50-1:35 システム概要とAI活用の工夫
- 「中央の図1が全体フローです。入力から字幕、要約、source-only QA、出力までを接続しています。」
- 「本体AIは Azure OpenAI / Azure AI Speech / Azure AI Search / LLM as a Judge です。」
- 「QAは `Use only lecture sources` を明示し、`source_id` 引用を必須にしています。」
- 「人間側では、fallback理由を確認し、プロンプトや閾値の採否を判断して改善しました。」

## 1:35-2:15 結果・評価
- 「表1に実測をまとめています。」
- 「字幕遅延2項目と字幕品質`5/5`は達成です。」
- 「QA E2E`9.18s`、ASR補正遅延`6.41s`、補正採択率`0/5`は未達または要改善です。」
- 「未達を隠さず、次の改善に繋げています。」

## 2:15-2:45 分析・考察
- 「字幕達成の要因は、短い処理経路と評価条件の明確化です。」
- 「QA未達の主因は、検索→回答→検証の直列処理と、回答生成タイムアウト時のfallbackです。」
- 「制約として、ASR補正採択率が低いこと、CER自動計測が難しいことがあります。」

## 2:45-3:00 今後の展開と締め
- 「改善順は3つです。1つ目がQA遅延短縮、2つ目がASR補正採択率改善、3つ目がCER半自動化です。」
- 「下部QRから、デモ動画、実働アプリ、GitHubを確認できます。」
- 「以上です。ご質問では、未達項目の原因と改善計画を中心にお答えします。」
