# F1 Azure OpenAI Summary Architecture

## Overview

Integrate Azure OpenAI for 30-second lecture summary generation in F1 Multimodal Lecture Assist, replacing the current deterministic text concatenation approach.

## Current State Analysis

### Existing F1 Summary (lecture_summary_service.py)
- **Approach**: Deterministic text concatenation
- **Method**: `_build_summary_text()` concatenates speech and OCR text
- **Evidence**: Fixed 3 speech + 3 visual events
- **Key Terms**: Simple token extraction from events
- **Limitations**: No semantic understanding, no intelligent summarization

### F4 QA Patterns to Reuse (lecture_answerer_service.py)
- **Azure OpenAI Integration**: `_call_openai()` pattern
- **Prompt Engineering**: Japanese prompts with structured guidelines
- **Error Handling**: `LectureAnswererError` with network/parse fallbacks
- **Configuration**: `azure_openai_*` settings from config.py

### Existing Constraints
- 600 character limit for summary text (`MAX_SUMMARY_CHARS`)
- 30-second windows (`WINDOW_SIZE_MS = 30_000`)
- Evidence types: speech, slide, board (`LectureEvidenceType`)
- API contract: `LectureSummaryLatestResponse` schema

## Japanese Prompt Template

### Summary Generation Prompt

```python
def _build_summary_prompt(
    self,
    speech_events: list[SpeechEvent],
    visual_events: list[VisualEvent],
    window_start_ms: int,
    window_end_ms: int,
) -> str:
    """Build Japanese prompt for 30-second lecture summary."""

    # Format sources
    speech_text = self._format_speech_sources(speech_events)
    visual_text = self._format_visual_sources(visual_events)

    prompt = f"""あなたは講義の30秒区間を要約するアシスタントです。

以下の講義資料（音声転写と視覚情報）を使用して、この区間の要約を作成してください。

【音声情報】
{speech_text}

【視覚情報（スライド・板書）】
{visual_text}

【時間範囲】
{window_start_ms // 1000}秒 〜 {window_end_ms // 1000}秒

要約の要件:
- 講義資料に基づいて要約してください（外部知識は使用禁止）
- 600文字以内で簡潔にまとめてください
- 重要なキーワードを最大5つ抽出してください
- 引用元（音声/スライド/板書）を明示してください
- 日本語で自然な文章で書いてください

出力形式（JSON）:
{{
  "summary": "要約文（600文字以内）",
  "key_terms": [
    {{"term": "キーワード1", "evidence_tags": ["speech"]}},
    {{"term": "キーワード2", "evidence_tags": ["slide", "speech"]}}
  ],
  "evidence_count": 3
}}

JSONのみを出力してください（説明不要）:"

    return prompt
```

### Evidence Tagging Rules

| Evidence Type | Source | Tag |
|--------------|--------|-----|
| Speech | `SpeechEvent.text` | `speech` |
| Slide | `VisualEvent.source == "slide"` | `slide` |
| Board | `VisualEvent.source == "board"` | `board` |

### Modified lecture_summary_service.py

```python
class SqlAlchemyLectureSummaryService:
    def __init__(
        self,
        db: AsyncSession,
        summary_generator: LectureSummaryGeneratorService | None = None,
    ) -> None:
        self._db = db
        self._summary_generator = summary_generator

    async def _build_window(
        self,
        *,
        session_id: str,
        window_end_ms: int,
        persist: bool,
    ) -> LectureSummaryLatestResponse:
        # ... fetch events ...

        # Use Azure OpenAI if available
        if self._summary_generator is not None:
            draft = await self._summary_generator.generate_summary(
                speech_events=speech_events,
                visual_events=visual_events,
                window_start_ms=window_start_ms,
                window_end_ms=window_end_ms,
            )
            summary_text = draft.summary
            key_terms = draft.key_terms
            evidence = draft.evidence
        else:
            # Fallback to deterministic (existing logic)
            summary_text = self._build_summary_text(...)
            key_terms = self._build_key_terms(...)
            evidence = self._build_evidence(...)
```

### Dependency Injection in API

```python
# app/api/v4/lecture.py
@app.post("/lecture/summary/latest")
async def get_latest_summary(
    session_id: str,
    user_id: str,
    db: AsyncSession = Depends(get_db),
    settings: Settings = Depends(get_settings),
):
    # Create Azure OpenAI summary generator if enabled
    summary_generator = None
    if settings.azure_openai_enabled:
        summary_generator = AzureOpenAILectureSummaryGeneratorService(
            api_key=settings.azure_openai_api_key,
            endpoint=settings.azure_openai_endpoint,
            model=settings.azure_openai_model,
        )

    service = SqlAlchemyLectureSummaryService(
        db=db,
        summary_generator=summary_generator,
    )
    return await service.get_latest_summary(session_id, user_id)
```

## Error Handling Strategy

### Azure Disabled State

**Requirement**: ERROR when Azure disabled (no deterministic fallback)

```python
async def generate_summary(
    self,
    *,
    speech_events: list[SpeechEvent],
    visual_events: list[VisualEvent],
    window_start_ms: int,
    window_end_ms: int,
) -> LectureSummaryDraft:
    """Generate summary with Azure OpenAI (required)."""

    if not self._is_azure_openai_ready():
        raise LectureSummaryGeneratorError(
            "Azure OpenAI is required for summary generation. "
            "Please configure azure_openai_enabled=true with valid credentials."
        )

    # ... proceed with Azure OpenAI call ...
```

### Error Classification

| Error Type | Handling | User Message |
|-----------|----------|--------------|
| HTTPError (4xx) | Raise error | Azure OpenAI API error |
| HTTPError (5xx) | Retry once | Temporary API error |
| URLError | Raise error | Network connection failed |
| JSONDecodeError | Raise error | Invalid API response |
| Empty response | Raise error | Empty summary generated |
| Timeout | Retry once | API timeout |

## Risk Analysis

### Risks and Mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| **Azure downtime** | High - summary unavailable | Clear error message, graceful degradation |
| **Slow API response** | Medium - UI latency | 30s timeout, async operation |
| **Token limit exceeded** | Low - truncated summary | Max tokens=800, char limit enforcement |
| **Invalid JSON response** | Medium - parse error | JSON schema validation, error handling |
| **High Azure costs** | Low - budget impact | Temperature=0.5, max_tokens=800 (controlled) |
| **Hallucination** | Medium - inaccurate summary | Source-only constraint in prompt |

## Key Design Decisions

| Decision | Rationale |
|----------|-----------|
| **Separate service class** | Clean separation of concerns; reusability; testability |
| **Protocol-based interface** | Flexibility for future implementations; mocking support |
| **JSON output from LLM** | Structured parsing; type safety; validation |
| **Source-only constraint** | Prevents hallucination; grounded answers |
| **Error on Azure disabled** | Clear requirement; no silent fallback to deterministic |
| **Reuse answerer patterns** | Consistency; reduced duplication; proven patterns |
| **Temperature=0.5** | Balance between creativity and consistency |
| **Max tokens=800** | Sufficient for 600 char Japanese; cost control |
