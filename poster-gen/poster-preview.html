SIT Copilotは「source-only QA + 引用 + 検証」を実装（講義前中後を接続）
2. 目的
講義の前・中・後を一貫支援し、聞き漏らしと非母語話者の理解負荷を下げる。
  • 対象: 留学生・聞き漏らしが多い受講者・アクセシビリティ配慮対象
  • 成果指標: 字幕遅延<3s、QA遅延<5s、字幕品質5/5
  • 未達を隠さず、改善順まで提示する
新規性: 字幕だけで終わらず、source-only QAで根拠付き復習に接続。

4. AI利用とプロンプト設計
AI利用の要点
  • 字幕: Azure AI Speech + Azure OpenAI（gpt-5-nano）
  • 字幕変換は運用記録として gpt-4.1-nano も利用
  • QA: Azure AI Search + Azure OpenAI + LLM as a Judge
  • OCR: Azure AI Vision、開発支援: Codex / Claude / Gemini
詳細設定・検証ログは GitHub と docs/performance-evaluation.md を参照
プロンプト設計の工夫（要点）
字幕変換
入力: text + target_lang_mode
制約: 専門用語保持 / 1発話1出力
切替: ja / easy-ja / en
QA（source-only）
Use only lecture sources.
`chunk_id` を必ず引用。
根拠不足は回答拒否。

6. 分析・次アクション
字幕が目標達成した理由
  • 字幕変換は単機能APIで計測し、処理経路を短く保てた
  • 3文×2モードで反復計測し、0.76s/0.99sを確認
QAが未達の理由
  • 検索→回答→Judge検証が直列でE2E遅延が積算
  • 現環境で回答生成タイムアウト→fallbackが発生
  • 検索/生成/Judgeの内訳計測は今後
次の3アクション
  • 1) QA内訳計測を実装し、遅延支配点を確定
  • 2) 回答生成と検証の時間予算・再試行を見直す
  • 3) ASR補正閾値とCER評価手順を改善
